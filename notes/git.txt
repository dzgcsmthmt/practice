git 笔记：
1、git分区
工作区 -----------  代码区
缓存区 -----------  git add. 加入了追踪但是还没有commit
版本区 -----------  git commit后  .git文件夹下的文件
----------------------------------------------------------------------------------------------
2、git diff
git diff 	//工作区和缓存区的不同
git diff --cached或者 --staged 		//缓存区和版本区的不同
git diff master 	//工作区和版本区的不同
----------------------------------------------------------------------------------------------
3、版本回退
git reset --hard HEAD^ 回退到上一个版本
git reset --hard HEAD～number 回退到上number个版本
git log查看commit id
git reset --hard commit id 回退到指定的版本
git reflog查看所有的commit id
----------------------------------------------------------------------------------------------
4、git reset HEAD --file
撤销操作，在add后commit前可以使用，commit后就不能使用这个撤销了
----------------------------------------------------------------------------------------------
5、git checkout --filename
撤销工作区的修改操作，在git add之前使用，工作区的修改内容不保存
----------------------------------------------------------------------------------------------
6、git rm filename
工作区的新建文件，通过git add添加到缓存区后，删除工作区的文件，可以使用rm把暂存区的文件删除掉
可以用过git checkout -- filename把工作区中的文件恢复
git rm -f filename
工作区的文件没有删除，通过-f可以把工作区的和缓存区的文件都删除
git rm --cached filename
只删除缓存区的文件，不会删除工作区的文件
----------------------------------------------------------------------------------------------
7、git checkout commit id filename
可以通过commit id恢复操作 与3的不同是3为版本回退，checkout只能回退文件
----------------------------------------------------------------------------------------------
8、分支
git branch  查看分支
git checkout 切换分支
git checkout -b  新建并切换到新建分支
git branch -d 删除分支
git push origin :branch name 提交删除分支
